

ros launch engine outline:

    the ros launch engine will give the user the ability to query
        * Currently running ros launches
        * Available ros launch files that can be launched
        * details about a specific ros launch file
        * Start a ros launch if the ros launch file is known
        * Kill a ros launch process

each ros launch will be represented by a ros launch class with an abc roslaunchBase:
    The roslaunch base will define the following
        * available args
        * launch (returns launch error)
        * kill (returns kill error) 
        * is running 
        * external launch? 
        * internal launch?
